บุญฟอนต์

บุญ คือ ชุดตัวอักษรตระกูลหนึ่ง (font family) ที่มุ่งแสดงผลการจัดเรียงตัวอักษรไทยให้ถูกต้องในเว็บบราวเซอร์ ตัวอักษรทุกตัวทั้งไทยและละตินนั้นวาดขึ้นมาใหม่โดยตั้งใจให้สามารถใช้งานเป็นฟอนต์ไทยตัวอ่านมีหัวที่ดูไม่แย่นักในขนาด 14px ขึ้นไป และมีความหนาใกล้เคียงกับตัวละติน

๏ แรงบันดาลใจ

บุญฟอนต์ ได้รับแรงบันดาลใจจากฟอนต์อื่นหลายตระกูล เช่น Roboto, Helvetica (Arial), Thonburi ใน Mac/iOS, Garuda ในชุด Fonts-TLWG และ TH Sarabun New (หรือ SP Laksaman ในชุด ThaiFonts-SiamPradesh ที่ปรับสเกลใหม่)

ตัวอักษรไทยใน บุญฟอนต์ จะเล็กกว่าฟอนต์ไทยทั่วไปเมื่อจับคู่กับตัวละติน (อย่างน้อยก็ฟอนต์ไทยใน Linux ที่ผมใช้งานอยู่) โดยตั้งใจกำหนด ความสูง บ. ให้ต่ำกว่า x-height เล็กน้อย เพราะการเขียนคำไทยนั้นใช้พื้นที่ในแนวตั้งมากกว่าคำภาษาอังกฤษ เพื่อให้ใช้งานฟอนต์ไทยในเว็บได้สบายตาโดยไม่ต้องกำหนด line-height เกิน 1.5em ใน CSS ตัวอักษรไทยไม่แน่นเกินไปและตัวอักษรละตินไม่ห่างเกินไปในแนวตั้ง ถ้าเข้าใจไม่ผิดโดยใช้สายตาวัด ฟอนต์ไทยของ iOS อย่าง Thonburi ก็แก้ปัญหาแบบนี้? โดยให้ สูง บ. ต่ำกว่า x-height เล็กน้อย นอกจากนี้บุญฟอนต์ยังมีความหนาของตัวอักษรไทยใกล้เคียงกับตัวละติน บุญตัวหนา จึงหนามากเมื่อเทียบกับฟอนต์ไทยตัวอื่นแต่ดูกลมกลืมเมื่อใช้คู่กับตัวละติน (ถ้าผมไม่คิดไปเองนะ ทดสอบกับ heading text โดยใช้บุญฟอนต์ตัวไทยจับคู่กับฟอนต์ serif อย่าง Georgia ด้วยตัวหนาแล้วก็ยังพอดูได้ เพราะความหนาไม่ต่างกันมาก)

บุญฟอนต์ เพิ่งจะเริ่มต้นพัฒนาและทดลองคุณสมบัติต่างๆ (และโทษสมบัติ) ยังห่างไกลจากคำว่าสมบูรณ์แบบ หวังพึ่ง Anti-Alias หรือ Text Renderer จากระบบปฏิบัติการเท่านั้น เพราะไม่ได้ Hint หรือทำ Grid Fitting บุญตัวหนา (Bold) ก็ยัง kern ได้แย่และเบียดกันมากเมื่อใช้ในขนาดเล็ก ส่วน บุญตัวเอน (Italic) ก็ใช้โปรแกรมทำอัตโนมัติ จึงดูขาดๆ เกินๆ

นอกจากนี้อาจมีการเปลี่ยนแปลงคุณสมบัติอีกหลายอย่างได้ทุกเมื่อทุกเวลาหลังจากทดสอบการใช้งานในระบบปฏิบัติการและบราวเซอร์ที่หลากหลายมากขึ้น เพราะแต่ละตัวมีความสามารถในการรองรับคุณสมบัติของ OpenType ไม่เท่ากัน โดยเฉพาะอย่างยิ่งเมื่อเจออักษรยูนิโค้ดที่ไม่ใช่อักษรละตินอย่างอักษรไทย ณ เวลาที่เขียน ผมทดสอบบุญฟอนต์ใน Ubuntu 13.04 ด้วย Chrome และ FireFox รุ่นล่าสุด และ Safari ใน iOS 6.1.3 เท่านั้น หากใครพบปัญหาหรือบั๊กในระบบปฏิบัติการและบราวเซอร์อื่น ก็แจ้งเข้ามาได้

บุญฟอนต์ แจกจ่ายและเปิดเผยซอร์สโค้ดด้วยหวังว่ามันจะมีประโยชน์และเป็นส่วนหนึ่งของการพัฒนาฟอนต์ไทยเพื่อให้เหมาะต่อการใช้งานทั้งในโปรแกรมทั่วไปและในเว็บไซต์ และโปรดทราบไว้ล่วงหน้าว่าคุณต้องยอมรับความเสี่ยงจากการใช้งานฟอนต์นี้เอง ผู้พัฒนาไม่ต้องรับผิดชอบต่อความเสียหายที่อาจจะเกิดจากการใช้งานฟอนต์นี้ ไม่ว่ากรณีใดๆ

๏ วิธีใช้งาน

คุณสามารถดาวน์โหลดไฟล์ฟอนต์และซอร์สโค้ดเวอร์ชั่นล่าสุดได้ที่ Boon tags page หรือโคลนทั้งโปรเจ็คต์ด้วย Git เพื่อดัดแปลงด้วยโปรแกรม FontForge ได้ที่ github.com/sungsit/boon

๑. การใช้งานกับโปรแกรมทั่วไป

ติดตั้งไฟล์ฟอนต์ประเภท OpenType จาก /fonts/otf/*.otf ณ เวลาที่เขียน ทดสอบใน Ubuntu 13.04 เท่านั้น ยังไม่ได้ทดสอบในระบบปฏิบัติการอื่น แต่น่าจะใช้งานได้ปกติกับระบบปฏิบัติการและโปรแกรมที่รองรับ OpenType

๒. การใช้งานในเว็บไซต์

ใช้ไฟล์ฟอนต์จาก /fonts/woff/*.woff ไฟล์ .woff ก็คือ TrueType นั่นเอง แต่ห่อและบีบให้ขนาดเล็กลงเพื่อให้เหมาะต่อการใช้งานในอินเตอร์เน็ต นั่นเป็นเหตุผลที่บุญฟอนต์ไม่มีไฟล์ .ttf เพราะไฟล์ .otf กับ .woff ก็เกินพอ (แถม .svg เผื่อใครไม่ถนัดใช้ FontForge)

การใช้ฟอนต์ไทยในเว็บไซต์นั้นจำเป็นต้องใช้ CSS และ โชค เข้าช่วย โชคในที่นี้หมายความว่าเว็บบราวเซอร์แต่ละตัวรองรับ OpenType Features ไม่เท่ากัน แม้จะใช้ @font-face ก็ตาม

ณ เวลาที่เขียน แม้จะยังไม่ได้ทดสอบแต่ก็แน่ใจว่า บุญฟอนต์ไม่สนับสนุนการใช้งานกับบราวเซอร์รุ่นเก่าหลายรุ่น ตัวอย่างเช่น Internet Explorer ก่อนรุ่น 9 และ mobile browser อีกหลายตัว เพราะถึงแม้ IE รุ่นเก่าจะสามารถใช้งาน @font-face กับไฟล์ .eot ได้ หรือบราวเซอร์ในอุปกรณ์เคลื่อนที่รุ่นเก่าจะเรียกใช้ฟอนต์จากไฟล์ .svg ได้ แต่เท่าที่ผมจำได้ไฟล์พวกนี้ไม่รองรับ opentype features นั่นเป็นเหตุผลที่วรรณยุกต์ไทยต้องลอยสูงไว้ก่อนเพราะมันเป็นตำแหน่งที่ปลอดภัยที่สุดในบราวเซอร์ที่แย่ที่สุด จริงๆ สามารถใช้งานฟอนต์ไทยทุกตัวกับ IE รุ่นเก่าได้ แต่คุณต้องแปลงฟอนต์ฟอร์แมตเองและต้องยอมรับด้วยว่าวรรณยุกต์ไทยต้องลอยอย่างหลีกเลี่ยงไม่ได้ จะได้ไม่โกรธฟอนต์หรือคนทำฟอนต์! (ผมคิดว่าสำหรับ IE รุ่นเก่านั้นแนะนำให้ผู้ใช้เปลี่ยนบราวเซอร์หรือติดตั้งฟอนต์ไทยเพิ่มในระบบปฏิบัติเองคงง่ายกว่าการแก้ปัญหาด้วยวิธีอื่น!)

คุณสามารถดูตัวอย่างการใช้ CSS กับบุญฟอนต์ได้จากซอร์สโค้ดของหน้าเว็บนี้ หรือจากไฟล์ /css/boon.css ที่สำคัญคือต้องใช้ text-rendering: optimizeLegibility และ/หรือ font-feature-settings ใน CSS ด้วย (โน้ต: text-rendering ไม่ใช่ CSS property และมีผลต่อประสิทธิภาพและความรวดเร็วในการ render ตัวอักษรของบราวเซอร์ - อ่านเพิ่มเติม) เพื่อบอกบราวเซอร์ที่ฉลาดพอให้เรียกใช้งานคุณสมบัติของ OpenType ที่ฟอนต์ตัวนั้นมี (smcp, kern, liga เป็นต้น) แม้บราวเซอร์รุ่นใหม่บางตัวจะเปิดไว้ให้โดยอัตโนมัติสำหรับตัวอักษรขนาดเกิน 20px แล้วก็ตาม

๓. Kerning ตัวอักษรละติน

อักษรละตินบางตัวต้องจัดการความห่างในแนวราบระหว่างกันเป็นพิเศษ เช่น ตัว T, A, V, W, Y ใหญ่ กับ ตัวอักษรเล็กอื่นๆ อีกหลายตัว เพื่อไม่ให้มันดูห่างกันมากเกินไป เว็บบราวเซอร์ส่วนใหญ่จะไม่เรียกใช้ Kerning feature จากฟอนต์จนกว่าจะระบุใน CSS ด้วย text-rendering: optimizeLegibility (โดยปกติตัวอักษรไทยไม่ต้อง kern ในแนวราบก็ได้ ยกเว้นสระหน้าตัวสูงอย่าง โ ใ และ ไ เพราะพยัญชนะแต่ละตัวใช้พื้นที่ว่างค่อนข้างใกล้เคียงกัน และมีตำแหน่งสระบน / สระล่าง / วรรณยุกต์ คอยกำกับระยะห่างเส้นหลังอยู่จึงไม่มีปัญหาแบบตัวละตินเล็ก-ใหญ่)

  • เพิ่มน้ำหนักและสไตล์ เช่น Light, Heavy, Black, Condensed (ถ้าเป็นไปได้)
  • เพิ่ม Small Cap Glyphs เวลาต้องใช้อักษรละตินตัวใหญ่ติดกันเยอะๆ จะได้ไม่ดูน่าเกลียดเกินไป
  • เพิ่ม Glyphs อักษรลาว ยังไม่ได้ศึกษาวิธีเขียนภาษาลาว แต่คาดว่าใกล้เคียงกับภาษาไทยมาก และตัวอักษรลาวก็มีน้อยกว่าอักษรไทย น่าจะทำฟอนต์ไว้ใช้งานคู่กันได้ (ไอเดียเพิ่มเติมที่คล้ายกันคืออักษรเขมร)
